# Savvy — Personal Finance & Investment Platform

Savvy is a multi-organization personal finance and investment platform with AI-powered insights, real-time market data, role-based access control, and end-to-end 2FA security. Built on React + Vite + Tailwind, with Lovable Cloud (Supabase) handling auth, database, and edge functions.

## Feature overview

- **Dashboard** — net worth, recent transactions, spending charts.
- **Expenses, Budgets, Savings goals** — full CRUD, currency-aware.
- **Portfolio & Market Data** — holdings, watchlist, real-time quotes, price alerts.
- **AI Insights (Savvy AI)** — concise, data-grounded financial advice with image/PDF analysis.
- **Tasks** — assignable workflow items per organization.
- **Organizations** — create or join via 8-character code, multi-member, role-gated.
- **Settings** — profile, currency, preferences, notifications, language, 2FA, recovery codes, audit log, user management (Owners only).
- **2FA / TOTP** — enrollment with QR + manual key, downloadable recovery codes, atomic single-use consumption.
- **Password reset** — secure email-based flow with strength meter on the reset page.
- **8 languages** — English, French, Spanish, Portuguese, Arabic (RTL), Chinese, Swahili, Kinyarwanda. Selection persists to the database.

## Tech stack

- **Frontend**: React 18, Vite 5, TypeScript 5, Tailwind CSS v3, shadcn/ui, react-router, react-i18next, TanStack Query.
- **Backend**: Lovable Cloud (managed Supabase) — Postgres + RLS, Auth, Edge Functions (Deno).
- **AI**: Lovable AI Gateway (`google/gemini-2.5-flash` by default).
- **Charts**: Recharts.

## Installation

```bash
# 1. Install dependencies
bun install   # or: npm install

# 2. Start dev server
bun run dev   # or: npm run dev
```

The app runs at `http://localhost:8080`.

> Lovable Cloud is provisioned automatically — no separate Supabase account needed. The `.env` file is auto-managed; do not edit it.

## Database & backend

Tables live in the `public` schema with Row-Level Security on every table:

- `profiles`, `user_settings` — per-user data.
- `organizations`, `organization_members`, `membership_requests` — multi-tenant org model.
- `holdings`, `watchlist`, `price_alerts` — investment data.
- `tasks` — workflow items.
- `notifications`, `settings_audit_log` — activity tracking.
- `mfa_recovery_codes` — single-use 2FA backup codes.

Edge functions in `supabase/functions/`:

- `ai-insights` — streams AI chat responses (concise mode).
- `market-quotes`, `exchange-rates` — market data proxies.
- `check-price-alerts` — cron-driven alert evaluator.

## Authentication & security

- **Email + password** signup/login (default).
- **Google OAuth** via `lovable.auth.signInWithOAuth`.
- **TOTP 2FA** — enroll in Settings → Security; verified factor required to reach AAL2.
- **Recovery codes** — 10 single-use codes generated on demand. Atomic consumption prevents reuse.
- **Sensitive actions** (disabling 2FA, etc.) require a TOTP or recovery-code challenge.
- **Password reset** — `supabase.auth.resetPasswordForEmail` → user lands on `/reset-password` with strength validation.
- **Back to homepage** link on Login/Signup for easy navigation.

## Role-based access control

Stored in `organization_members.role` (enum `app_role`):

| Role | Capabilities |
|---|---|
| `owner` | Full admin: manage members & roles, view/regenerate join code, all data CRUD. |
| `accountant` | Approve/reject membership requests, create/edit tasks, all financial CRUD. |
| `analyst` | Read + analytics, limited writes. |
| `viewer` | **Read-only**. UI hides write actions; banner indicates read-only mode. |

`OrganizationContext` exposes `isOwner`, `isViewer`, `canEdit` for UI gating. The User Management Panel (Owners only) shows each member's name, **email**, and role with inline role changes and removal (with confirmation + audit log).

The organization **join code** is visible only to Owners.

## AI Insights

`/ai-insights` streams responses from `ai-insights` edge function. The system prompt enforces:

- ≤ 6 short bullets or 4 short sentences.
- Lead with the single most important insight.
- Use specific numbers from user data.
- No filler or restating the question.

Supports text, receipt images, and PDF text extraction.

## Localization

Powered by `react-i18next`. Locale files in `src/i18n/locales/`. Switching language via the sidebar selector:

1. Updates the UI **instantly**.
2. Persists `preferences.language` to `user_settings` for the logged-in user.
3. Reapplies on next login (any device).

Arabic automatically switches the document to RTL.

## Troubleshooting

**Login redirects back to /auth**
The session may be stale. Clear cookies/localStorage and retry.

**"Invalid or already used recovery code"**
Each code works exactly once. Regenerate a new batch from Settings → Security if you've exhausted them.

**2FA enrollment shows "factor exists"**
Stale unverified factors are auto-cleaned on the next enroll attempt — just click *Enable* again.

**AI Insights says "Rate limit exceeded" (429)**
Lovable AI Gateway throttle. Wait 30 seconds and retry.

**AI Insights says "Credits exhausted" (402)**
Add credits in **Settings → Workspace → Usage** in the Lovable dashboard.

**Language doesn't persist after logout**
Make sure you're logged in when switching — the language is saved per user in `user_settings.preferences.language`.

**Membership request stuck on "Pending"**
Only the Owner or an Accountant in that organization can approve or reject. Contact an admin.

**Reset email not arriving**
Check spam. Default Lovable Cloud emails are used; delivery can take 1–2 minutes. Verify the email exists on a registered account.
